## 📖 文档维护流程

### 更新接口文档

```
1. 修改接口代码
   ↓
2. 更新 Swagger 注解
   ↓
3. 运行检查脚本: npm run docs:check
   ↓
4. 更新 API接口文档.md
   ↓
5. 在变更记录中添加更新内容
   ↓
6. 提交代码
```

详见 [接口文档维护指南.md](./接口文档维护指南.md)

## 💡 小贴士

### 如何快速查找接口

在 [API接口文档.md](../API接口文档.md) 中：
- 使用编辑器的搜索功能
- 按模块分类查找
- 查看目录快速导航

### 如何确认最新接口

1. 启动服务后访问: `http://localhost:3000/api/docs`
2. Swagger 会根据代码注解自动生成最新文档
3. 如发现 Markdown 文档与 Swagger 不一致，以 Swagger 为准

### 如何提交文档更新

提交时，请确保：
- API接口文档.md 已同步更新
- 接口文档变更记录.md 已添加记录
- 文档版本号已更新（如需要）
