# 接口文档使用说明

## 🎯 目标读者

本文档是为以下人员准备的：

1. **前端开发人员** - 需要调用 API 接口
2. **后端开发人员** - 需要维护接口文档
3. **测试人员** - 需要验证接口功能

---

## 📖 快速开始

### 对于前端开发人员

#### 1. 获取接口文档

打开 `D:\selfProject\API接口文档.md`，这是**最新的接口文档**。

#### 2. 查看接口详情

文档中包含了所有接口的详细信息：
- **接口地址**：如何调用接口
- **请求参数**：需要发送什么数据
- **响应示例**：接口返回什么数据
- **认证方式**：是否需要登录
- **错误码**：可能的错误及含义

#### 3. 测试接口

**开发环境**：
```bash
基础 URL: http://localhost:3000
完整接口路径: http://localhost:3000/api/v1/<接口路径>
```

**获取 Token**：
```javascript
// 使用 login 接口获取
const response = await fetch('http://localhost:3000/api/v1/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'test@example.com',
    password: 'password123'
  })
})
```

**使用 Token**：
```javascript
const response = await fetch('http://localhost:3000/api/v1/users/profile', {
  headers: {
    'Authorization': 'Bearer <your-token>'
  }
})
```

#### 4. 常见问题

**Q**: 接口返回 401 (Unauthorized) 怎么办？
**A**: Token 过期或无效，请重新登录获取新 Token。

**Q**: 接口返回 403 (Forbidden) 怎么办？
**A**: 无权限访问此接口，请检查用户角色和权限。

### 对于后端开发人员

#### 1. 更新接口代码

在 `D:\selfProject\backend\src\modules\` 中修改接口。

#### 2. 更新文档

接口代码修改后，运行以下命令检查是否需要更新文档：

```bash
cd D:\selfProject\backend
npm run docs:check
```

**如果检查通过**：
```
✅ API接口文档.md 已同步更新
```

**如果检查失败**：
```
❌ API接口文档.md 未更新！
请参考: D:\selfProject\docs\接口文档维护指南.md
```

#### 3. 快速打开文档

```bash
cd D:\selfProject\backend
npm run docs:open
```

#### 4. 更新步骤

详细步骤请阅读：[接口文档维护指南.md](./docs/接口文档维护指南.md)
