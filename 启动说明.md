# 项目启动说明

## 快速启动（推荐）

### 1. 使用 Docker 启动（推荐）

```bash
# 1. 确保 Docker 和 Docker Compose 已安装

# 2. 启动所有服务
cd D:\selfProject
docker-compose up -d

# 3. 等待服务启动完成后，运行数据库迁移
docker-compose exec backend npx prisma migrate dev

# 4. 初始化种子数据（可选）
docker-compose exec backend npm run prisma:seed

# 5. 访问 API 文档
# 打开浏览器访问: http://localhost:3000/api/docs
```

### 2. 本地开发（需要 Node.js 18+）

```bash
# 1. 安装依赖
cd backend
npm install

# 2. 配置环境变量
# 复制 .env.example 为 .env 并配置正确的值
cp .env.example .env

# 3. 确保 PostgreSQL 和 Redis 服务运行

# 4. 数据库迁移
npx prisma migrate dev

# 5. 初始化数据（可选）
npm run prisma:seed

# 6. 启动开发服务器
npm run start:dev

# 7. 访问 API 文档
# http://localhost:3000/api/docs
```

## 常见问题

### 1. 数据库连接失败
- 确保 PostgreSQL 服务正在运行
- 检查 .env 文件中的数据库连接配置
- 如果使用 Docker，请确保容器正在运行

### 2. 端口被占用
- 修改 docker-compose.yml 中的端口映射
- 或停止占用对应端口的服务

### 3. 依赖安装失败
```bash
# 清理 npm 缓存
npm cache clean --force

# 删除 node_modules 并重新安装
rm -rf node_modules package-lock.json
npm install
```
